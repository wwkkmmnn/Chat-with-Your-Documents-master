# Chat-with-Your-Documents 启动说明

本文档适用于当前仓库：

- 后端：`FastAPI`
- 前端：`React + Vite`
- LLM API：`DeepSeek API`
- 前端地址：`http://localhost:5173`
- 后端地址：`http://localhost:8000`

注意：

- 当前代码里前端请求地址写死为 `http://localhost:8000`
- 后端 CORS 只放行了 `http://localhost:5173`
- 所以本项目默认必须按这两个端口启动

## 1. 环境要求

你当前机器里已经有：

- `Node.js v22.14.0`
- `npm 10.9.2`
- `Python 3.12.x`

理论上可以尝试直接运行，但这个项目后端依赖里包含 `faiss` 和句向量模型，Windows 下如果安装失败，建议优先改用 `Python 3.10` 或 `Python 3.11`。

## 2. 项目结构

仓库根目录：

- `backend_chatdoc.py`：后端入口
- `front-chatdoc/`：前端项目
- `uploaded_documents/`：运行后自动生成，保存上传的 PDF
- `faiss_vector_store/`：运行后自动生成，保存向量库

## 3. 启动后端

在仓库根目录打开 PowerShell：

```powershell
cd D:\git\Chat-with-Your-Documents-master
```

### 3.1 创建虚拟环境

```powershell
python -m venv .venv
```

### 3.2 激活虚拟环境

```powershell
.\.venv\Scripts\Activate.ps1
```

如果提示脚本执行被禁止，可以临时执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

### 3.3 安装后端依赖

这个仓库没有提供 `requirements.txt`，可以先手动安装当前代码实际用到的依赖：

```powershell
pip install fastapi "uvicorn[standard]" python-dotenv openai langchain langchain-community langchain-huggingface langchain-text-splitters sentence-transformers pypdf python-multipart faiss-cpu
```

如果你想顺手升级 `pip`，可以先执行：

```powershell
python -m pip install --upgrade pip
```

### 3.4 配置环境变量

在项目根目录新建 `.env` 文件，内容如下：

```env
DEEPSEEK_API_KEY=你的_deepseek_api_key
DEEPSEEK_MODEL=deepseek-chat
```

说明：

- 没有这个 key，后端大概率无法正常回答问题
- 文档上传和向量化流程可以启动，但问答阶段会依赖 DeepSeek API
- 如果你想切换模型，也可以把 `DEEPSEEK_MODEL` 改成 `deepseek-reasoner`

### 3.5 启动后端服务

方式一，直接运行脚本：

```powershell
python .\backend_chatdoc.py
```

方式二，使用 `uvicorn` 热重载：

```powershell
uvicorn backend_chatdoc:app --host 0.0.0.0 --port 8000 --reload
```

看到类似下面的信息，说明后端启动成功：

```text
Uvicorn running on http://0.0.0.0:8000
```

## 4. 启动前端

新开一个 PowerShell 窗口，执行：

```powershell
cd D:\git\Chat-with-Your-Documents-master\front-chatdoc
```

### 4.1 安装前端依赖

```powershell
npm install
```

### 4.2 启动前端开发服务器

```powershell
npm run dev
```

正常情况下会看到类似输出：

```text
Local:   http://localhost:5173/
```

然后在浏览器打开：

```text
http://localhost:5173
```

## 5. 正确的启动顺序

建议按下面顺序：

1. 先启动后端 `8000`
2. 再启动前端 `5173`
3. 打开前端页面
4. 上传 PDF
5. 等上传完成后再提问

## 6. 使用流程

页面打开后：

1. 上传一个 PDF 文件
2. 后端会读取 PDF、切分文本、生成向量并保存到本地
3. 在输入框里输入问题
4. 前端调用 `/query/` 接口
5. 后端先检索相关文本，再调用 Together 模型生成答案

## 7. 首次运行时的现象

首次运行时可能会比较慢，原因包括：

- 需要安装 Python 依赖
- `sentence-transformers/all-mpnet-base-v2` 可能会首次下载模型
- 首次创建 `faiss_vector_store` 目录和索引文件

## 8. 常见问题

### 8.1 前端能打开，但上传或提问失败

通常先检查：

- 后端是否真的运行在 `http://localhost:8000`
- 前端是否运行在 `http://localhost:5173`
- `.env` 里是否配置了 `DEEPSEEK_API_KEY`

### 8.2 浏览器报跨域错误

当前后端只允许：

```text
http://localhost:5173
```

如果你不是这个地址启动前端，就会跨域失败。

### 8.3 `faiss-cpu` 安装失败

这是 Windows 环境里比较常见的问题。如果遇到：

- 安装报错
- Python 版本不兼容
- 编译失败

建议优先尝试：

1. 换用 `Python 3.10` 或 `Python 3.11`
2. 重新创建虚拟环境
3. 再执行依赖安装

### 8.4 上传成功了，但回答不正常

可能原因：

- 没有配置 DeepSeek API Key
- PDF 内容提取失败
- 向量库里混入了之前上传过的旧文档

注意：这个项目当前会把多次上传的内容合并进同一个向量库，不会自动按文件隔离。
